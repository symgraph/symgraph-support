# SymGraph Support

This repository is the public support intake for the hosted SymGraph web service.

If you are a SymGraph customer and need help with the web app, API, account access, workspace behavior, or a service-side regression, open a GitHub issue here.

## Use This Repo For

- Bugs or unexpected behavior in the SymGraph web application
- API errors or service-side failures
- Account, login, or workspace issues that can be described publicly
- Service regressions, degraded behavior, or outage reports
- General usage questions about the hosted SymGraph service

## Before You Open An Issue

Please do not post sensitive material in public issues, including:

- API keys, passwords, session tokens, or other credentials
- Proprietary binaries, private datasets, or customer-confidential documents
- Unredacted logs, screenshots, or stack traces that expose private data

If you need to share examples, sanitize them first.

## How To File A Good Support Ticket

Include as much of the following as you can:

- What happened
- What you expected to happen
- Steps to reproduce the issue
- When it happened, including timezone if relevant
- Your organization or workspace name, if the issue is scoped to one
- Any relevant endpoint, page, browser, or client workflow involved
- Sanitized screenshots, request IDs, or error messages

Clear, reproducible reports get resolved faster.

## Issue Types

When opening an issue, make the title and description clear about which category it falls into:

- Bug report
- Account or access issue
- Service degradation or outage report
- Usage or support question

If the problem is production-impacting, say so clearly in the title and first paragraph.

## Response Expectations

Issues in this repo are monitored and triaged as part of SymGraph support.

Please include enough context in the initial report to avoid back-and-forth. If we need more detail, we will follow up in the issue.

## Product-Specific Issues

This repository is for the hosted SymGraph web service.

If your issue is specific to a client plugin or MCP server such as GhidrAssist, BinAssist, or IDAssist, we may redirect you to the appropriate product repository when that is the better place to track it.
